## Part 2 — Fix the matching generator

The goal: each matching card cell must contain a visible character image (sticker) from the library.

### How the generator should work after the fix

1. Read `assets/stickers/matching/index.json`
2. Look up the puzzle theme (from `activity.json` `theme` field, normalized to lowercase) against the themes map
3. If theme found: pick N sticker names from that theme's array (N = number of pairs, typically 6)
4. If theme not found: fall back to `animals` theme
5. For each pair: load `assets/stickers/matching/{theme}/{name}.png`
   - If file exists: use it as card content image
   - If file missing: draw a colored shape fallback (existing behavior)
6. Draw each card with the sticker image centered and scaled to fit within the card bounds

### matching.json contract — do not change the shape

The existing contract (read from sample) must be preserved. Only the rendered PNG output changes (cards now have content). The `matching.json` sidecar shape does not change.

### Joyo in matching (challenge phase)

Once cards have content, add Joyo using `joyo_thinking.png` during the challenge phase (kids are trying to remember where the pairs are before the solve reveal). Position: same pattern as OC-020/021 — lower-right of puzzle area, idle bob. Switch to `joyo_celebrating.png` at end of solve. This is the same `PuzzleJoyoLayer` pattern from OC-020, just `thinking` instead of `running` during challenge.

Add `'matching'` as a handled type in `PuzzleJoyoLayer` (in `ActivityChallenge.jsx`):
- Challenge phase: `joyo_thinking.png` at lower-right, idle bob
- Solve end: `joyo_celebrating.png` (already the default from OC-020)

## Part 3 — Solve reveal for matching

Currently matching uses `WipeReveal` (left-to-right wipe from blank to solved). This works as a fallback but a matched-pairs reveal should feel different.

If the matching generator produces `matchRects` (positions of each card pair) in `matching.json`, implement a `MatchingReveal` component that flips cards one by one (or pair by pair) from face-down to face-up over the solve phase.

If `matchRects` are not in the current `matching.json` contract: keep `WipeReveal` for now. Do NOT redesign the generator data contract in this task — note it as a follow-up.
